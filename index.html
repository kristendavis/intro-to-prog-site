Accessing a Website

  1. you navigate to website on browser
  2. it is sent over the internet using
  3. HTTP (Hypertext Transfer Protocol) to...
  4. Servers which then send the site to your browser

Attributes in HTML

  • Attributes generally require a closing tag.
  • Those that do not are called "void."
  • A completed attribute is referred to as an "element."
  • Attributes can be inline or block.

Some Inline Attributes

  • a href (links to a website)
  • img (embeds an image)
  • strong, em, b (text formatting)
  • br (line break)

Some Block Attributes

  • p (paragraph)
  • span
  • div

Language Roles

  • HTML (HyperText Markup Language) = Structure
  • CSS (Cascading Style Sheets) = Style
  • JavaScript = Interactive Tools, ie; garage door opener

Google Developer Tools

  • not all elements are visible
  • all elements are rectangular
  • everything is there down to the text that appears on the page

Page Structure

  • the tree-like structure allows the programmer to more easily visualize the elements in the page
  • DOM (Document Object Model) refers to the tree-like structure of programs
  • HTML tags are made into elements in the DOM or "tree"
  • HTML defines structure and content
  • CSS defines the look
  • CSS structure seems to start with larger, most basic elements and progress through the smaller, more specific pieces of a site

CSS (Cascading Style Sheets)

  • Rules in CSS start at the most specific and work their way down in priority
  • In other words, the most specific overrules
  • The rule that best describes the element is applied to that element
  • CSS Style Order
    • default style browser
    • separate style sheet file
    • stylesheet inside HTML (NOT recommended except in small projects)
    • inline style element (avoid if possible)
  • Cascading means that the rules are applied to elements that directly match and all child elements
  • Most specific rules take effect in the event of overlapping rules

Box Model

  • Content: Very center, image or content
  • Padding: clears and area around the content
  • Border: goes around padding and content
  • Margin: clears area around border, no background color

Image Credit: w3.org

Code, Test, Refine

  • First, get the structure right by "boxifying"
    • Look for natural boxes
    • Look for repeated styles and semantic elements
  • Write your HTML
    • tags (headings, etc)
  • Apply styles (biggest to smallest)
    • Size
    • Positioning
  • Then focus on smaller details
  • Lastly, go back and fix things

Stage 2

Command Line

  • use GitBash to navigate command line for course purposes
  • different commands in command line include
    • ls - list contents of current directory
    • cd - "change directory" to another one in your current directory
    • cd.. - move up to the parent directory
    • pwd - print working directory, tells you where you are
    • start (open on a Mac) - to open a currently existing file or folder
    • mkdir - "make directory", creates a new folder in your current directory
    • touch - creates a new file in your current diredtory (you need to use the suffix to name the file type), this will also update a currently existing file's timestamp
    • rm - "remove" to permanently delete a file
    • rmdir - "remove direcctory" this will only delete empty directories
    • rm -rf - "remove with recursive force" BEWARE, it will permanently delete the directory along with any files and folders inside
  • you can do multiple actions of the same type at once by placing a space between the items you want to perform the action on, for example rm first.txt second.txt third.txt would delete all three of those files. You can also do this with mkdir
  • for our purposes in this course, use your CLI to run your Python code. Do this by navigating to the directory and then typing python[name_of_file]
  • it will run the code and tell you about any errors

Python

  • Backus Naur Form (a way of writing something and replacing until you come to all terminal items)
  • expression operator expression - (with Python) expressions can be replaced with expressions or numbers, operators include (+-*/)
  • in Python variables are created simply by making the variable name equal to its desired value

Variables in Python

  • can be used to store numbers or strings
  • you assign a variable a value by typing the name of your variable, an equals sign and the value that you want your variable to hold
  • when Python evaluates that assignment it considers the right side first, and assigns it to the variable name you provided on the left
  • the equals sign is used to store the value not to denote equality as in a math problem
  • strings are denoted by '' or ""
  • "2" is a string and 2 is a number

Indexing Strings

  • an index on a string is a location
  • the index starts with 0 and each symbol in the string is assigned a number counting up from there
  • you can use indexing to find specific items in a string or to remove or add to a string
  • indexing can also be useful in capitalization and in finding an entire string within the search area

Finding and Replacing

  • string.find("thing_to_find", start_here) returns the location of thing_to_find. Start here can be a number or a variable defined with a number that tells the program where to start looking in the string
  • string.replace("thing_to_find", "new_thing") replaces thing_to_find with new_thing. You can set a max number of characters as a third parameter if you want

Functions

  • input is what you tell the computer
  • output is what the computer returns
  • functions in python start like this: function_name(parameters, parameters)
  • if you want a function to do anything outside of itself you have to include a return statement
  • anything after the last return statement will not be evaluated

Equality Comparisons

  • less than(<), greater than(>), less than/equal to(<=), greater than/equal to(>=), not equal to(!=), equal to(==)
  • output Boolean value: True or False

If Statements

  • always place a colon after the if or the else statment to indicate the beginning of the if statment execution code
  • indention indicates whether code is included in the if else statement
  • you can use or when checking for two different True possibilities
  • with or the computer checks the first condition and if it is true the second condition is not evaluated
  • if the first condition of the or statement is false, the value of the second condition is the value of the whole statement
  • you can nest if statments

While Loops

  • set up similarly to if statements but iterates until its condition is fulfilled
  • this can lead to infinite loops
  • break statements give us a way to exit a loop even while it is still true
  • this can help prevent infinite loops
  • break statements are if statements with a condition that uses bread to exit the loops

Debugging

Examining Error Messages

  • in Python these are called tracebacks
  • they give you line numbers and error types
  • you can google these to find solutions

Checking Intermediate Results

  • a good strategy when working from example code is to copy and paste and then make changes one at a time, checking for errors along the way
  • then when you have a working example of what you want, compare it to your first try (which should be commented out for clarity's sake)
  • when bugs do not create error message it can be harder to locate the problem
  • printing things along the way can help to narrow down the portion of your code that is causing problems

Keep and Compare Old Versions

  • it's better to try different versions in the same file and comment out the ones you are not currently working on
  • the comparison that you get with this method can help you to see your errors

Writing Comments

  • don't comment on self-explanatory items
  • start functions and classes with a comment about the expected inputs and outputs as well as the purpose of the funcion or class
  • update comments as you go when things change
  • keep comments as concise as possible (if this is difficult your process within the code could probably use some work, your code should not depend on comments for readability)

Structured Data

  • a string is a type of structured data - a sequence of characters, e.g. 'udacity'
  • a list is a sequence of anything, e.g. p = ['u','d','a','c','i','t','y']
  • lists can be indexed like strings
  • you can nest lists within lists
  • with nested lists and indexing the following is possible: you can access the second item in the second list like this: list[1][1] where the first number corresponds to the index of the second list in a list and the second number corresponds to the second item in that second list

Mutation

  • you cannot use mutation with a string, strings that are changed are simply reassigned to a different value, never changed
  • using an index assignment (e.g. list[0]) and assigning it a value will replace that index's value

Aliasing

  • two variable names assigned to the same object means that any change to one will also change the other one
  • one alias can be reassigned to a different value and the two are no longer aliases

List Operations

  • append - list.append(element) - toadd a new element to the end of the list
  • + - [0,1] + [2,3] gives [0,1,2,3] - kind of like concatenating two lists
  • len - len(list) - gives the number of elements in the list, it only counts the outer elements not any lists within the list
  • index - list.index(value) - will find the value in the list and return that value's index, but returns an error when the value is not found
  • in - value in list - returns a Boolean of True if value is there and False if it is not
  • value not in list - gives the opposite Booleans

For Loops

  • for item in list:
  • set up similarly to while loops
  • they automatically iterate through each item in the list
  • the item is not a counter! Do not use it like this

Understanding a Computational Problem

  • problems have inputs and outputs
  • the problem is defined by all possible inputs (set, usually infinite) and the relationship between inputs and desired outputs
  • Pythonista's Guide to All Problems in the Galaxy
    • Step 0: DON'T PANIC
    • Step 1: Understand the INPUTS
    • Step 2: Understand the OUTPUTS
    • Step 3: Understand the I/O relationship by trying some examples
    • Step 4: Consider how to systematically solve the problem
    • Step 5: Develop incrementally and TEST along the way
  • don't optimize prematurely
  • try to make it simple and correct
  • a good strategy is to use "pseudo code" to make an algorithm
    • go through step by step and write down the "code" you would enter for each step
    • look at what you have and try to identify edge cases that are not handled by the current solution
